## 摘要
歧义性对作为机器人规划器的大语言模型（LLMs）构成了重大挑战。在本文中，我们提出了场景图思维链（SG-CoT），这是一个两阶段框架，其中 LLM 迭代地查询环境的场景图表示，以检测并澄清歧义。首先，根据输入观测构建环境的结构化场景图表示，捕捉物体、其属性以及与其他物体的关系。其次，LLM 配备了检索函数，用于查询场景图中与给定指令相关的部分。这使 LLM 的推理过程基于观测，从而提高了机器人在歧义情况下的规划可靠性。SG-CoT 还允许 LLM 识别歧义来源，并向用户或其他机器人提出相关的消歧问题。广泛的实验表明，SG-CoT 一贯优于先前的方法，在提问准确率上至少提高了 10%，在单智能体环境中的成功率至少提高了 4%，在多智能体环境中至少提高了 15%，验证了其在更具泛化性的机器人规划中的有效性。

## Abstract
Ambiguity poses a major challenge to large language models (LLMs) used as robotic planners. In this letter, we present Scene Graph-Chain-of-Thought (SG-CoT), a two-stage framework where LLMs iteratively query a scene graph representation of the environment to detect and clarify ambiguities. First, a structured scene graph representation of the environment is constructed from input observations, capturing objects, their attributes, and relationships with other objects. Second, the LLM is equipped with retrieval functions to query portions of the scene graph that are relevant to the provided instruction. This grounds the reasoning process of the LLM in the observation, increasing the reliability of robotic planners under ambiguous situations. SG-CoT also allows the LLM to identify the source of ambiguity and pose a relevant disambiguation question to the user or another robot. Extensive experimentation demonstrates that SG-CoT consistently outperforms prior methods, with a minimum of 10% improvement in question accuracy and a minimum success rate increase of 4% in single-agent and 15% in multi-agent environments, validating its effectiveness for more generalizable robot planning.

#### 1. 核心问题与整体含义
在机器人领域，大语言模型（LLM）常被用作高层规划器，但它们面临一个核心挑战：**歧义性（Ambiguity）**。这种歧义可能源于用户指令的不明确（如“给我拿点吃的”），也可能源于环境状态（如场景中有多个相同的杯子，或根本没有杯子）。现有的 LLM 规划器往往过于自信，且缺乏对环境细节（如物体属性、空间关系）的深度理解，容易导致错误或危险的决策。本文提出了 **SG-CoT** 框架，旨在通过结构化的场景图（Scene Graph）和迭代的思维链（CoT）推理，使机器人能够主动识别并澄清环境及指令中的歧义。

#### 2. 方法论
SG-CoT 采用两阶段流程：
*   **场景图构建（Stage 1）：** 利用视觉语言模型（VLM，如 Qwen3-VL）和物体检测器（如 Grounding DINO），将环境观测转化为结构化的场景图 $G=(V, E)$。节点 $V$ 代表物体及其属性（颜色、状态等），边 $E$ 代表物体间的语义或空间关系（如“在...左边”、“在...里面”）。输出为易于解析的 JSON 格式。
*   **迭代推理与检索（Stage 2）：** 
    *   **工具调用：** LLM 不再是一次性生成计划，而是通过 API 函数（`retrieve_node` 和 `retrieve_edge`）迭代地查询场景图。
    *   **接地推理（Grounded Reasoning）：** LLM 先生成推理轨迹（Thought），决定需要了解哪些环境细节，调用 API 获取结果，再根据返回的真实观测修正理解。
    *   **歧义处理：** 如果发现多个匹配对象（Multiplicity）、对象缺失（Absence）或指令模糊（Underspecification），LLM 会停止执行并生成针对性的澄清问题（如“你是要红色的那个还是蓝色的那个？”）。
    *   **多智能体协作：** 在部分可观测环境下，机器人可以通过 `ask_robot` 动作向其他机器人查询其视野内的场景图信息。

#### 6. 主要结论与发现
*   **性能提升：** SG-CoT 在提问准确率（CQR）上比最优基准方法提升了至少 **10%**。
*   **成功率：** 在单智能体任务中成功率提升了 **4%** 以上，在多智能体任务中提升了 **15%**。
*   **迭代的重要性：** 消融实验显示，迭代检索比一次性输入整个场景图效果更好，因为迭代过程能有效减少 LLM 的幻觉并防止错误累积。
*   **模型规模正相关：** 使用更强大的模型（如 Gemini）时，SG-CoT 的表现显著优于较小的模型（如 Qwen3-2B）。

#### 7. 优点
*   **深度接地：** 通过场景图将 LLM 的抽象推理与具体的物理属性、空间关系紧密结合。
*   **主动消歧：** 不仅仅是检测不确定性，还能通过查询场景图“定位”歧义的具体原因（如发现两个红杯子），从而提出高质量的问题。
*   **通用性：** 能够处理环境诱导的歧义，而不仅仅是文本层面的语义模糊。
*   **多智能体支持：** 扩展了机器人在部分观测下的通信推理能力。

#### 8. 不足与局限
*   **感知依赖：** 框架高度依赖 VLM 构建场景图的准确性。如果 VLM 在第一阶段产生幻觉（如识别错物体关系），会直接误导后续规划。
*   **成本与延迟：** 迭代推理和长上下文（历史记录拼接）导致 Token 消耗大，推理延迟较高，可能影响实时性要求极高的任务。
*   **实验局限：** 目前仅在**模拟环境**中验证，缺乏真实物理世界（Real-world）的实验数据，未考虑传感器噪声和动态环境的影响。
*   **可扩展性：** 随着环境物体数量剧增，场景图的构建和检索深度可能会面临效率瓶颈。
