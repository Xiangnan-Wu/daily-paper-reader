## 摘要
视觉语言导航（VLN）模型表现出卓越的导航精度，但计算开销巨大。Token 缓存已成为一种极具前景的免训练策略，通过重用 Token 计算结果来降低成本；然而，现有的 Token 缓存方法依赖于视觉域方法进行可缓存 Token 的选择，在适配 VLN 模型时面临挑战：1) 当存在视点迁移时，视觉域方法会失效；2) 在没有额外算法辅助的情况下，视觉域方法会忽略关键的边缘信息；3) 视觉域方法忽视了场景的时间变化，且缺乏缓存预算的可调节性。在本文中，我们进行了详细分析，发现这些挑战的影响在频率域中表现出不变性和可分析性。基于此，我们提出了一种名为 FreqCache 的频率引导 Token 缓存框架。利用频率域的固有特性，FreqCache 实现了最优的 Token 缓存建立、更新和自适应调整。实验表明，FreqCache 在开销可忽略的情况下实现了 1.59 倍的加速，展示了在 VLN Token 缓存中集成频率域方法的效果。

## Abstract
Vision-Language-Navigation (VLN) models exhibit excellent navigation accuracy but incur high computational overhead. Token caching has emerged as a promising training-free strategy to reduce this cost by reusing token computation results; however, existing token caching approaches rely on visual domain methods for cacheable token selection, leading to challenges when adapted to VLN models. 1) Visual domain methods become invalid when there is viewpoint migration. 2) Visual domain methods neglect critical edge information without the aid of additional algorithms. 3) Visual domain methods overlook the temporal variation of scenarios and lack adjustability in cache budgets. In this paper, we develop detailed analyses and find that the impacts of these challenges exhibit invariance and analyzability in the frequency domain. Based on these, we propose a frequency-guided token caching framework, called FreqCache. Utilizing the inherent properties of the frequency domain, FreqCache achieves optimal token cache establishment, refreshment, and adaptive adjustment. Experiments show that FreqCache achieves 1.59x speedup with ignorable overhead, showing the effect of integrating frequency domain methods in VLN token caching.

### 1. 核心问题与整体含义
视觉语言导航（VLN）模型虽然在复杂任务中表现卓越，但其巨大的计算开销（尤其是基于大语言模型的 VLA 架构）限制了在机器人上的实时部署。**Token 缓存（Token Caching）** 是一种通过重用相邻时间步相似视觉 Token 来减少冗余计算的技术。

然而，现有的 Token 缓存方法主要基于**视觉域（Visual Domain）**，在 VLN 任务中面临三大挑战：
*   **视点迁移（Viewpoint Migration）：** 机器人移动导致图像像素位移，视觉域的逐位置匹配会失效，导致不必要的缓存刷新。
*   **边缘感知缺失（Edge Unawareness）：** 视觉域方法难以识别关键的障碍物边缘，错误地重用边缘 Token 会导致机器人发生物理碰撞。
*   **时间变化（Temporal Variation）：** 导航场景复杂度随时间动态变化，静态的缓存预算无法兼顾效率与安全。

本文提出的 **FreqCache** 框架旨在通过**频率域（Frequency Domain）**分析来解决上述问题，实现高效且安全的导航加速。

### 2. 方法论
FreqCache 的核心思想是利用频域特性的不变性和可分析性来指导 Token 的选择与重用。其包含三个核心模块：

*   **模块 I：视点迁移感知 Token 缓存建立**
    *   利用傅里叶变换的**幅值谱不变性**。即使图像发生平移，幅值谱也保持稳定。
    *   通过**相位相关法（Phase Correlation）**计算相邻帧间的精确空间位移 $(\Delta i, \Delta j)$，从而对齐特征图，识别出真正可重用的区域，减少因视点移动导致的重复计算。
*   **模块 II：边缘信息相关 Token 识别与刷新**
    *   利用频域的**正交解耦特性**。背景能量集中在低频，而关键边缘（如门框、障碍物）在特定高频段有显著能量响应。
    *   通过分块离散余弦变换（DCT）和高通滤波器提取每个 Patch 的高频能量，强制刷新能量激增的 Token，确保导航安全性，防止碰撞。
*   **模块 III：时间自适应缓存预算确定**
    *   引入**频谱熵（Spectral Entropy）**作为场景复杂度的内在指标。
    *   简单场景（如走廊）频谱熵低，增加缓存比例；复杂场景（如杂乱房间）频谱熵高，减少缓存以保证感知精度。通过指数函数动态映射频谱熵到缓存预算 $K_{reuse}$。
*   **系统实现：** 采用 GPU 多进程并行设计，将 VLN 推理、频域处理和 Token 选择分配到不同的计算流中，利用 CUDA Core 和 Tensor Core 协同工作，掩盖频域处理的开销。
