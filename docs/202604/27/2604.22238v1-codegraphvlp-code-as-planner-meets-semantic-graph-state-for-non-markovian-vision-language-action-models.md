## 摘要
视觉-语言-动作（VLA）模型有望实现通用机器人操控，但通常作为短时域策略进行训练和部署，这些策略假设最新的观测结果足以进行动作推理。这一假设在非马尔可夫长时域任务中失效，因为在这些任务中，与任务相关的证据可能会被遮挡或仅出现在轨迹的早期阶段，且杂乱的环境和干扰物会使细粒度的视觉定位变得脆弱。我们提出了 CodeGraphVLP，这是一个分层框架，通过将持久的语义图状态与可执行的基于代码的规划器以及进度引导的视觉-语言提示相结合，实现了可靠的长时域操控。语义图在部分可观测性下维护与任务相关的实体及其关系。合成的规划器在语义图上执行，以进行高效的进度检查，并输出子任务指令及与子任务相关的物体。我们利用这些输出构建抑制杂乱的观测结果，使 VLA 执行器专注于关键证据。在真实世界的非马尔可夫任务中，CodeGraphVLP 相比于强大的 VLA 基线和支持历史信息的变体，提高了任务完成率，同时与 VLM 在环（VLM-in-the-loop）规划相比，显著降低了规划延迟。我们还进行了广泛的消融研究，以确认各组件的贡献。

## Abstract
Vision-Language-Action (VLA) models promise generalist robot manipulation, but are typically trained and deployed as short-horizon policies that assume the latest observation is sufficient for action reasoning. This assumption breaks in non-Markovian long-horizon tasks, where task-relevant evidence can be occluded or appear only earlier in the trajectory, and where clutter and distractors make fine-grained visual grounding brittle. We present CodeGraphVLP, a hierarchical framework that enables reliable long-horizon manipulation by combining a persistent semantic-graph state with an executable code-based planner and progress-guided visual-language prompting. The semantic-graph maintains task-relevant entities and relations under partial observability. The synthesized planner executes over this semantic-graph to perform efficient progress checks and outputs a subtask instruction together with subtask-relevant objects. We use these outputs to construct clutter-suppressed observations that focus the VLA executor on critical evidence. On real-world non-Markovian tasks, CodeGraphVLP improves task completion over strong VLA baselines and history-enabled variants while substantially lowering planning latency compared to VLM-in-the-loop planning. We also conduct extensive ablation studies to confirm the contributions of each component.

现有的视觉-语言-动作（VLA）模型通常遵循**马尔可夫假设**，即认为当前的观测结果足以决定下一步动作。然而，在真实的长程（Long-horizon）机器人任务中，这一假设往往失效（即**非马尔可夫场景**）：
*   **信息缺失**：关键证据可能在轨迹早期出现后被遮挡或移出视野。
*   **环境干扰**：杂乱的背景和无关物体（Clutter）会干扰 VLA 模型的视觉定位，导致动作错误。
*   **效率瓶颈**：现有的分层模型（VLM 规划 + VLA 执行）依赖频繁调用大模型进行在线规划，导致高延迟，难以满足实时控制需求。

### 2. 方法论
CodeGraphVLP 提出了一种分层框架，核心思想是利用**结构化语义图**作为持久记忆，并通过**一次性生成的代码规划器**实现高效推理。

*   **持久化语义图状态 (Semantic-Graph State)**：
    *   **初始化**：利用实例分割（YOLOE）和 VLM 识别任务相关物体，通过几何和语义特征进行多视角关联，并根据空间关系诱导物体间的关联（如 `in`, `on`, `near`）。
    *   **在线更新**：使用 Cutie 跟踪器实时更新物体掩码，并定期通过分割模型修正漂移或发现新物体。
*   **代码即规划器 (Code-as-Planner)**：
    *   在任务开始前，利用 LLM（如 GPT-4）根据初始场景和任务指令生成一段 Python 代码。
    *   该代码作为高层规划器，通过查询语义图来检查任务进度（利用 `graph.task_memory` 记录已完成步骤），并输出当前的子任务指令及相关物体列表。
*   **进度引导的去干扰提示 (Clutter-Free Prompting)**：
    *   **视觉层面**：根据规划器输出的相关物体，对原始图像进行掩码处理，黑化所有无关像素，生成“去干扰”观察结果。
    *   **语言层面**：将长程指令简化为当前步骤的短促、明确的子任务指令。
    *   **执行**：VLA 模型（如 $\pi_0$）基于去干扰的视觉和语言输入预测动作。

### 3. 实验设计
*   **场景与任务**：在真实世界的 UR10e 机器人平台上设计了三个具有非马尔可夫依赖的任务：

    2.  **Place-and-Stack**：将方块放入杯中（方块随后被遮挡），再用另一个杯子叠放。
    3.  **Swap Cups**：在三个盘子间交换两个杯子的位置（涉及中间缓存位和顺序约束）。
*   **对比基线 (Benchmarks)**：

    *   **历史增强型**：支持多帧历史输入的 Gr00T N1.5。
    *   **消融变体**：不带去干扰视觉提示的 CodeGraphVLP、基于 VLM 在线规划的变体。

### 5. 实验数量与充分性
*   **实验规模**：收集了 400 场远程操作演示数据用于微调。
*   **消融实验**：针对“代码规划器的效率”和“去干扰提示的有效性”进行了专门的对比实验。

### 6. 主要结论与发现
*   **性能提升**：CodeGraphVLP 在三个任务上的平均成功率达到 **81.7%**，远高于最强基线（56.7%）。
*   **延迟降低**：相比于每步调用 VLM 的规划方式，Code-as-Planner 将规划延迟从约 **3 秒/步 降低至 0.3 秒/步**，提升了近 10 倍。
*   **鲁棒性**：去干扰视觉提示显著增强了模型在杂乱环境下的动作推理能力，消融实验显示去除该模块后成功率从 85% 降至 40%。

### 7. 优点
*   **高效性**：通过代码执行代替大模型在线推理，解决了分层架构的实时性难题。
*   **显式状态维护**：语义图提供了比非结构化历史记录（如 Memory Tokens）更清晰、更持久的任务上下文。
*   **视觉聚焦**：创新的去干扰提示方法有效缓解了 VLA 模型在复杂场景下的视觉过载问题。

### 8. 不足与局限
*   **依赖底层模型**：语义图的构建质量高度依赖于初始分割模型和 VLM 的属性识别能力，若初始化错误，后续规划将失效。
*   **代码合成风险**：LLM 生成的代码可能存在逻辑漏洞或无法执行的情况，虽然文中提到是一次性生成，但在复杂多变的任务中可能需要更强的自纠错机制。
*   **应用限制**：目前主要针对桌面操控任务，对于需要动态发现大量新物体的开放世界场景，语义图的维护成本和复杂度会显著增加。
