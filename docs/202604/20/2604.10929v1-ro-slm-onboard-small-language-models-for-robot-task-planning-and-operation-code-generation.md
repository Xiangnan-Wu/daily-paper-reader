## 摘要
近期大语言模型（LLMs）的进展赋予了机器人理解人类指令的上下文推理能力。然而，目前支持 LLM 的机器人通常依赖云端模型或高性能计算基础设施，这限制了它们在网络环境不可靠或计算资源受限的机器人（如无人机和小型地面车辆）上的部署。因此，部署支持机载部署的微调小语言模型（SLMs）提供了一个极具前景的替代方案。本文介绍了 Ro-SLM，这是一个通过蒸馏 LLMs 的知识和推理能力来实现可靠的 SLM 驱动机器人操作的框架。Ro-SLM 从数据集合成开始，利用 LLMs 生成多样化的任务指令，在极少人工协助的情况下产生相应的标准答案代码，并将指令增强到现实应用场景中。随后使用该数据集对 Ro-SLM 进行微调，其中 LLM 作为奖励函数来指导训练。在无人机操作任务上的广泛实验表明，Ro-SLM 将 SLM 的性能从无法支持机器人任务规划和代码生成，提升到了接近 LLM 的水平。

## Abstract
Recent advances in large language models (LLMs) provide robots with contextual reasoning abilities to comprehend human instructions. Yet, current LLM-enabled robots typically depend on cloud-based models or high-performance computing infrastructure, which limit their deployment on robots under unreliable internet environments or with constrained computational resources, such as UAVs and small ground vehicles. Thus, deploying fine-tuned small language models (SLMs) that support onboard deployment offers a promising alternative. This paper introduces Ro-SLM, a framework that enables reliable SLM-driven robot operation by distilling LLMs' knowledge and reasoning. Ro-SLM starts from dataset synthesis by leveraging LLMs to generate diverse task instructions, produce corresponding ground truth code with minimal human assistance, and augment instructions into real-world application scenarios. Ro-SLM is then fine-tuned with the dataset, in which LLM serves as a reward function to guide the training. Extensive experiments on UAV operation tasks demonstrate that Ro-SLM improves the performance of SLM from being incapable of supporting robotic task planning and code generation to achieving performance that approaches LLM.

### 2. 方法论：核心思想与关键技术
Ro-SLM 框架分为两个主要阶段：**数据集合成**与**模型微调优化**。

*   **阶段一：高质量数据集合成**
    1.  **任务指令生成 ($F_1$)**：利用 LLM 结合专家策略和少量示例，自动生成涵盖不同复杂度的机器人操作指令。
    2.  **标准答案代码生成 ($F_2$)**：采用“纠错式代码生成”框架。对于简单任务由 LLM 自动生成；对于涉及 5 步以上动作的复杂任务，引入**极少量人工辅助**进行校验，确保监督数据的绝对准确。
    3.  **场景增强 ($F_3$)**：利用 LLM 的世界知识，将枯燥的指令（如“向上飞5米”）改写为人类自然的描述（如“检查仓库顶部的通风口”），提高模型的鲁棒性。
    4.  **去注释处理**：移除代码中的所有注释，防止 SLM 过度拟合语言模式而非逻辑结构。

*   **阶段二：SLM 微调与优化**
    1.  **监督微调 (SFT)**：使用 LoRA 技术在合成数据集上进行初步对齐。
    2.  **基于 LLM 反馈的强化学习 (GRPO)**：使用“群体相对策略优化”算法。核心创新在于**将 LLM 配置为奖励函数**，它不比较文本相似度，而是解释代码背后的机器人行为逻辑，判断 SLM 生成的代码在功能上是否与标准答案等效。

### 6. 主要结论与发现
*   **性能飞跃**：原始 Llama-3.1-8B 几乎无法完成任务（成功率极低），而 Ro-SLM 使其在基础任务上达到 **97.7%** 的成功率，在复杂任务上达到 **70%**，接近甚至在某些极高难度任务上超过了 GPT-4o-mini。
*   **数据质量胜过数量**：通过人工辅助纠正复杂任务的 Ground Truth，SLM 能够学习到 LLM 容易出错的长程逻辑。
*   **去注释的重要性**：实验发现，保留注释会导致 SLM 性能下降，因为小模型容易被多样的语言模式干扰，移除注释能让其更专注于代码逻辑。

### 7. 优点与亮点
*   **机载可行性**：真正解决了“最后一公里”的部署问题，让机器人脱离云端独立思考。
*   **LLM 奖励函数**：巧妙地利用 LLM 作为逻辑评判者而非简单的文本匹配工具，解决了代码语义等效性评价的难题。
*   **纠错式合成**：通过 LLM 自动生成+关键步骤人工干预，以极低的人力成本构建了高质量的机器人指令数据集。

### 8. 不足与局限
*   **显存压力**：虽然是 SLM，但 8B 模型仍需约 16GB 显存，对于更小型的嵌入式设备（如 1B/3B 需求场景）尚未验证。

*   **人工依赖**：虽然号称“极少人工”，但在迁移到全新领域时，仍需专家编写 API 策略和校验复杂代码，扩展性受限。
*   **安全风险**：论文提到模型生成的代码直接控制物理硬件，若发生幻觉可能导致碰撞，需额外的安全护栏机制。
